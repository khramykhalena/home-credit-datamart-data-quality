# Home Credit Data Mart & Data Quality

Проект показывает, как сырые кредитные таблицы превращаются в аналитическую витрину для BI и системной аналитики.

## Цель
Подготовить аналитическую витрину по клиентам и заявкам, проверить качество данных и показать,
как сырые таблицы Home Credit превращаются в основу для отчётности, BI-дашборда и системного анализа процесса заявки.

## Что сделано
- анализ структуры входных таблиц и ключей связи;
- проверка качества данных: пропуски, дубликаты, аномальные значения, согласованность ключей;
- Python-пайплайн сборки витрины уровня `1 строка = 1 заявка`;
- агрегаты по кредитной истории из `bureau.csv`;
- агрегаты по истории прошлых заявок из `previous_application.csv`;
- подготовлена документация по полям витрины;
- добавлен SQL-эскиз для сборки аналогичной витрины в DuckDB/PostgreSQL.

## Текущие входные файлы
Обработаны:
- `HomeCredit_columns_description.csv`
- `application_train.csv`
- `bureau.csv`
- `previous_application.csv`

Поддерживается дополнительно, если вы добавите в `data/raw/`:
- `installments_payments.csv`

## Как запустить

### 1. Установить зависимости
```bash
pip install -r requirements.txt
```

### 2. Положить исходные CSV в папку `data/raw/`
Минимальный набор:
- `HomeCredit_columns_description.csv`
- `application_train.csv`
- `bureau.csv`

### 3. Собрать отчёт по качеству данных
```bash
python src/data_quality_report.py --data-dir data/raw --output-dir outputs
```

### 4. Собрать витрину
```bash
python src/build_datamart.py --data-dir data/raw --output-dir data/processed
```

По умолчанию скрипт создаёт:
- `data/processed/credit_application_datamart.parquet`
- `data/processed/credit_application_datamart.csv` (опционально, если указать `--save-csv`)

## Основная идея витрины
Базой является `application_train.csv`:
- одна строка = одна текущая заявка;
- ключ = `SK_ID_CURR`.

Сверху к заявке достраиваются агрегаты по кредитной истории из `bureau.csv`, например:
- количество внешних кредитов;
- количество активных и закрытых кредитов;
- сумма внешнего долга;
- сумма внешних кредитных лимитов;
- число кредитов с просрочкой;
- отношение долга к сумме кредитов.

Отдельным слоем добавляются агрегаты по `previous_application.csv`, например:
- количество прошлых заявок клиента;
- число прошлых одобрений и отказов;
- доля одобренных заявок;
- суммарные суммы прошлых заявок, кредитов и аннуитетов.

Если в проект добавить `installments_payments.csv`, пайплайн автоматически расширит витрину признаками платёжной дисциплины.
