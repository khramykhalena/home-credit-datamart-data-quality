# Анализ качества данных и подготовка аналитической витрины по кредитным заявкам

Проект показывает, как сырые таблицы Home Credit превращаются в аналитическую витрину для BI, отчётности и системного анализа кредитного процесса.

## Цель проекта

Подготовить витрину данных по клиентам и заявкам, проверить качество исходных таблиц и показать, как данные Home Credit можно привести к удобному аналитическому слою для:

- управленческой отчётности;
- BI-дашборда;
- сегментации клиентов;
- системного анализа процесса заявки;
- дальнейших исследовательских и скоринговых моделей.

## Что сделано

В рамках проекта:

- проанализирована структура входных таблиц и ключи связи;
- выполнены проверки качества данных: пропуски, дубликаты, аномалии, согласованность ключей;
- реализован Python-пайплайн сборки витрины уровня **1 строка = 1 заявка**;
- добавлены производные признаки по текущей заявке;
- добавлены агрегаты по внешней кредитной истории из `bureau.csv`;
- добавлены агрегаты по истории прошлых заявок из `previous_application.csv`;
- предусмотрено расширение витрины признаками из `installments_payments.csv`;
- подготовлена документация по полям витрины;
- добавлен SQL-эскиз сборки аналогичной витрины в DuckDB / PostgreSQL.

## Итог проекта

Ключевой результат проекта — аналитическая витрина уровня **1 строка = 1 заявка**, где базовая информация по текущей заявке объединяется с агрегатами по внешней кредитной истории и истории прошлых заявок.

Скрипт сборки сохраняет итоговую витрину в:

- `outputs/credit_application_datamart.parquet`
- или `outputs/credit_application_datamart.csv`

> На текущий момент в репозитории загружены результаты проверки качества данных. Финальный файл витрины можно дополнительно выгрузить в `outputs/`, чтобы итог проекта был виден прямо на GitHub.

## Ключевые результаты quality-check

По результатам анализа:

- `application_train` содержит **307 511 строк** и **122 колонки**;
- дублей по `SK_ID_CURR` в `application_train` нет;
- доля `TARGET = 1` составляет **8.07%**;
- значение `DAYS_EMPLOYED = 365243` встречается **55 374** раз и интерпретируется как техническая аномалия;
- `bureau` содержит слой связи формата **1:N**, в среднем **5.61** записей на клиента;
- доля активных внешних кредитов в `bureau` — **36.74%**;
- в `previous_application` среднее число прошлых заявок на клиента — **4.93**;
- доля прошлых заявок со статусом `Approved` — **62.07%**.
