# Анализ качества данных и подготовка аналитической витрины по кредитным заявкам

Проект показывает, как сырые кредитные таблицы Home Credit превращаются в аналитическую витрину, отчётности и системного анализа процесса заявки.

## Цель проекта

Подготовить аналитическую витрину по клиентам и заявкам, проверить качество данных и показать, как исходные таблицы Home Credit превращаются в основу для:

- управленческой отчётности;
- BI-дашборда;
- системного анализа кредитного процесса.

## Что сделано

В рамках проекта выполнены следующие задачи:

- проанализирована структура входных таблиц и ключи связи между ними;
- проведена проверка качества данных:
  - пропуски,
  - дубликаты,
  - аномальные значения,
  - согласованность ключей;
- реализован Python-пайплайн сборки витрины уровня **1 строка = 1 заявка**;
- добавлены агрегаты по кредитной истории из `bureau.csv`;
- добавлены агрегаты по истории прошлых заявок из `previous_application.csv`;
- подготовлена документация по полям витрины;
- добавлен SQL-эскиз для сборки аналогичной витрины в DuckDB / PostgreSQL.

## Использованные данные

### Обработанные входные файлы

- `HomeCredit_columns_description.csv`
